主机 IP 角色

所有虚拟机用户 root，密码 1。 由于使用本地 hostPath 持久化，所有有状态服务（MySQL/Redis/Nacos/Minio）通过 nodeAffinity 固定在 k8s-node01 上。

MySQL

创建 mysql.yaml：

 1apiVersion: apps/v1
 2kind: Deployment
 3metadata:
 4  name: ruoyi-mysql
 5  namespace: ruoyi
 6spec:
 7  replicas: 1
 8  selector:
 9    matchLabels:
10      app: ruoyi-mysql
11  template:
12    metadata:
13      labels:
14        app: ruoyi-mysql
15    spec:
16      affinity:
17        nodeAffinity:
18          requiredDuringSchedulingIgnoredDuringExecution:
19            nodeSelectorTerms:
20            - matchExpressions:
21              - key: kubernetes.io/hostname
22                operator: In
23                values:
24                - k8s-node01
25      containers:
26      - name: mysql
27        image: mysql:8.0.33
28        env:
29        - name: MYSQL_ROOT_PASSWORD
30          value: "ruoyi123"
31        ports:
32        - containerPort: 3306
33          name: mysqlport
34        volumeMounts:
35        - name: mysql-data
36          mountPath: /var/lib/mysql
37      volumes:
38      - name: mysql-data
39        hostPath:
40          path: /data/k8s/ruoyi/mysql
41          type: Directory
42---
43apiVersion: v1
44kind: Service
45metadata:
46  name: ry-db-svc
47  namespace: ruoyi
48spec:
49  ports:
50  - port: 3306
51    targetPort: 3306
52    nodePort: 32306
53  selector:
54    app: ruoyi-mysql
55  type: NodePort

⚠️ 注意：Service 名称为 ry-db-svc，这是 Nacos 和微服务中引用的数据库地址。

1kubectl apply -f mysql.yaml

导入数据库：

使用 Navicat 等工具通过 192.168.20.127:32306 连接 MySQL（密码 ruoyi123），创建 ruoyi 数据库并导入 SQL：

导入若依项目 sql/ 目录下的全部 4 个 SQL 文件。

⚠️ 注意事项：

  1. 属性前缀：新版本 Spring Cloud Gateway（5.x）使用 spring.cloud.gateway.server.webflux 前缀，而非旧版的 spring.cloud.gateway。如果不使用正确前缀，路由配置虽然能加载到 Spring 环境中，但无法被 GatewayProperties 绑定识别，所有路由将返回 404。
  2. CacheRequestBody 参数：CacheRequestBody 过滤器必须指定 bodyClass: java.lang.String，否则启动时会报 bodyClass must not be null。

常见问题排查

登录时报 404 “No static resource auth/login”

原因：Gateway 的 spring.cloud.gateway.* 路由配置属性前缀不匹配。

新版本 Spring Cloud Gateway（5.x）使用 spring.cloud.gateway.server.webflux 作为配置前缀，而非旧版的 spring.cloud.gateway。

修复：在 Nacos 中将配置前缀改为新版本格式，然后重启 gateway：

1kubectl rollout restart -n ruoyi deploy/ruoyi-gateway

Gateway 报 “bodyClass must not be null”

原因：CacheRequestBody 过滤器缺少 bodyClass 参数。

修复：将 - CacheRequestBody 改为带参数的形式：

1filters:
2  - name: CacheRequestBody
3    args:
4      bodyClass: java.lang.String

注意事项

  1. 镜像拉取密钥：从私有 Harbor 拉取镜像需要创建 harbor-secret：

    1kubectl create secret docker-registry harbor-secret \
2  -n ruoyi \
3  --docker-server=www.chenyang-helloworld.top \
4  --docker-username=<你的Harbor账号> \
5  --docker-password=<你的Harbor密码>

  2. 节点绑定：所有有状态服务（MySQL/Redis/Nacos/Minio）通过 nodeSelector 绑定到 k8s-node01，确保数据持久化不漂移。

  3. Nacos 配置更新：修改 Nacos 中的配置后，对应服务需要重启才能生效。

  4. Auth 服务不需要数据库：auth 模块排除了 DataSourceAutoConfiguration，只做认证逻辑。

  5. 健康检查：建议为每个 Deployment 添加 livenessProbe 和 readinessProbe，确保 K8s 能正确处理 Pod 异常。
